Q1: What is the key difference between map, forEach, filter, find function in JavaScript?

Ans:

map():map() method use for an array. map() returns a new array from an existing array element.
example: const newArray = existingArray.map((item) => item*2)

forEach():forEach() method also use for an array. forEach() doesn't return a new array from an existing array element.
example: existingArray.forEach((item) => console.log(item))

filter():filter() method use for an array. filter() returns a new array from an existing array element where the element satisfies the condition.
example: const newArray = existingArray.filter(item => item%2)

find():find() method use for an array. find() return a first element from an existing array element that satisfies the condition.
example: const element = existingArray.find(item => item%2)

Q2: What is the key difference between Local Storage, Session Storage and Cookies in JavaScript?

Ans:

Local Storage:Local Storage is a web storage method that helps us store data on the client’s computer in the form of key/value pairs in a web browser. The data is stored in local storage for a lifetime unless the user manually deletes it from the browser. It does not expire even when the user closes the window or tab.

Session Storage:Session Storage is very similar to the local storage. Still, the main difference lies in the lifespan as it persists in the browser until its current tab is on. Once you close the tab or terminate it, the data on session storage also gets lost.

Cookies:A cookie is an amount of information that persists between a server-side and a client-side. A web browser stores this information at the time of browsing. A cookie contains the information as a string generally in the form of a name-value pair separated by semi-colons.
